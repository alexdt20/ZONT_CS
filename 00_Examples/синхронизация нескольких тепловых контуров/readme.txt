
Сценарий "Синхронизация тепловых контуров" позволяет изменением уставки теплового контура сделанным на одном контроллере управлять уставками на других контроллерах, 
а так же синхронизовать отображение уставки на этом контроллере, если на других контроллерах были изменены уставки этого контура.

Синхронизироваться может любое количество контроллеров.
Контроллеры могут синхронизовывать заданную температуру контура по интерфейсу RS485 или UDP.
Прошивка контроллеров для выполнения этого скрипта должна быть не ниже 330.

Возможные применения скрипта:

1.  Если на объекте несколько контроллеров и некоторые контуры разных контроллеров должны управляться от одной уставки.

2.  Если на объекте несколько контроллеров, а связь с сервером есть только у одного. 
В этом случае можно сделать так, чтобы контуром отопления управлял контроллер, не имеющий связи с сервером. Или наоборот - контуром на контроллере, не имеющем связи, управлял контроллер, который имеет связь с сервером. 
Для этого на контроллере, с которого предполагается управлять, настраивается виртуальный контур ("пустышка") задача которого только синхронизироваться с другим контроллером.
Пример:
В бане стоит контроллер без связи (Баня). Домом управляет контроллер со связью (Дом).
Контроллеры подключены по RS485. На контроллере Дом создаем виртуальный контур потребителя «Отопление бани *». На контроллере Баня создаем реальный контур потребителя с таким же названием «Отопление бани *».
Таким образом, с контроллера Дом можно менять уставку на контроллере Баня. А если к контроллеру Баня подключена локальная панель управления и уставка на контроллере Баня будет изменена с этой панели 
– то на контроллере Дом будет отображено это изменение. 

Для синхронизации в контроллеры должны быть загружены скрипты:
hc_sync.pcl
utils.pcl

!!! Кроме того, на всех контроллерах должны быть настроены контуры отопления с одинаковым названием заканчивающимся символом *
